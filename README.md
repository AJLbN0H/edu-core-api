# EduCore API: Система управления образовательным контентом

EduCore API — это высокоуровневая Backend-платформа, разработанная в качестве дипломного проекта. Сервис предназначен для организации учебного процесса, хранения материалов и строгого контроля доступа на основе ролевой модели.

## Ключевые архитектурные решения

### 1. Трехуровневая ролевая модель (RBAC)
В отличие от стандартных решений, здесь реализована расширенная модель User с явным разделением на роли:
* Администратор — полный контроль над системой.
* Преподаватель — управление разделами и материалами (создание, редактирование своего контента).
* Студент — доступ к просмотру обучающих материалов.

### 2. Автоматизированный Identity Management
При создании пользователя через API система автоматически:
* Хеширует пароль.
* Назначает соответствующую роль.
* Добавляет пользователя в системные группы Django (Groups), что позволяет масштабировать права доступа через стандартную админ-панель.

### 3. Инкапсуляция логики доступа
Права доступа вынесены в кастомные пермишены (permissions.py). Логика `IsAdminOrTeacherOwner` гарантирует, что преподаватель может изменять только те материалы, владельцем которых он является, в то время как администратор сохраняет глобальный доступ.

### 4. Интерактивная документация
Проект полностью документирован с использованием Swagger и ReDoc. Это позволяет фронтенд-разработчикам сразу интегрироваться с API, видя все эндпоинты, типы данных и требования к авторизации.

## Технологический стек

* База: Python 3.13, Django 5.2.
* API: Django REST Framework (DRF).
* Безопасность: JWT (SimpleJWT), CORS Headers, Django Groups.
* Документация: drf-yasg (Swagger).
* База данных: PostgreSQL.
* Качество кода: Black (linting), Coverage (тестирование).

## Установка и запуск

1. Клонируйте репозиторий:
   ```git clone https://github.com/AJLbN0H/edu-core-api.git```

2. Подготовка окружения:
   Установите зависимости через Poetry:
   ```poetry install```
   Или через pip:
   ```pip install -r requirements.txt```

3. Настройка переменных:
   Создайте файл .env на основе .env.sample. Обязательно укажите параметры подключения к PostgreSQL и секретный ключ Django.

4. База данных:
   ```python manage.py migrate```

5. Запуск платформы:
   ```python manage.py runserver```

## API Эндпоинты (основные)
* /users/register/ — регистрация новых пользователей.
* /materials/sections/ — управление разделами обучения (ViewSet).
* /materials/items/ — управление конкретными материалами (ViewSet).
* /swagger/ — интерактивная документация.
